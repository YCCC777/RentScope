Privacy Policy

RentScope · Last updated April 2026

Summary

RentScope collects no personal data and sends nothing to any server. All processing happens entirely within your browser. There are no analytics, no tracking pixels, no third-party SDKs, and no network requests to any external service from this extension.

What we access

Page content (zillow.com only)

RentScope reads the DOM of Zillow listing pages to extract rental information (price, bedroom count, ZIP code). This data is used locally to look up HUD FMR data and display the overlay. It is never transmitted off your device.

Browser local & session storage

RentScope stores the current listing data in chrome.storage.session (cleared when you close the browser) so the popup can display it. No data is stored in chrome.storage.sync or sent to any cloud service.

Bundled HUD data

The FMR data is bundled inside the extension package (fmr_index.json, ~2–3 MB). It is read locally — no network request is made to HUD or any other server at runtime.

What we do NOT do

  • No data is collected, stored remotely, or shared
  • No browsing history or search queries are tracked
  • No analytics or crash reporting services are used
  • No user accounts or sign-in required
  • No ads, no third-party scripts, no remote code execution

Permissions explained

storage

Used for chrome.storage.session to pass the current listing data between the content script and popup within the same browser session. Data is cleared when the browser is closed.

tabs

Used to detect whether the active tab is a Zillow listing page, so the popup can show the right content. The extension does not read tab content beyond what the content script on zillow.com already sees.

Host permission: zillow.com

Required to inject the content script on Zillow listing pages. The extension only runs on zillow.com/apartments/* and zillow.com/homedetails/*.
